This is my final project for the CS50 web programming course. It is a workout diary app that tries to respond to some of the challenges I have faced with more complex tracking apps.

User Problems to solve

  • Too much complexity in creating a plan for a workout
  • Lack of flexibility in adjusting a workout once a workout has started
  • Lack of intelligent switching between weight units to accomodate use of different machines in different gyms
  • Inability to quickly and easily see history with an exercise, including PBs
  • Inability to track my weight within the same app - a key reason why I work out
  • Too much clutter with features I don't want, getting in the way of ones I do.

Requirements

The app tries to solve each of these challenges, with these basic requirements:

  1. Register/Login/Logout functions
  2. See a list of workout plans the user previously created
  3. Create a new workout plan
  4. Add new exercises that can be included in workout plans
  5. See a list of exercises the user has created
  6. View current personal bests for the exercises, and click in to see personal best history
  7. Select a workout plan to do as a workout that day
  8. Adjust a workout during the workout e.g. if a machine is not available
  9. Adjust the weight units for a given exercise
  10. Break a set into two or more, if the required rep count is to high at the set weight
  11. See a list of all the workouts a user has done, and when
  12. Add a new weight for the user to track
  13. See a chart of the user's weight over time

  15. Exit animation on save new plan in create workout plan
  16. Broken sets can only go one level deep
  17. Animation on expanding section in the list of workouts
  18. Update an exercise
